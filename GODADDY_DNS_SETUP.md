# Настройка DNS на GoDaddy для Resend

## Пошаговая инструкция

### 1. Получите DNS записи из Resend

1. Зайдите в Resend: https://resend.com/domains
2. Нажмите **"+ Add Domain"**
3. Введите: **faloclaro.com**
4. Resend покажет список DNS записей, которые нужно добавить

### 2. Войдите в GoDaddy

1. Откройте https://www.godaddy.com
2. Войдите в аккаунт
3. Перейдите в **My Products**
4. Найдите домен **faloclaro.com**
5. Нажмите на **"DNS"** или **"Manage DNS"**

### 3. Добавьте DNS записи

#### A. SPF запись (TXT)

1. Нажмите **"Add"** в разделе **TXT Records**
2. Заполните:
   - **Name/Host:** `@` (или оставьте пустым)
   - **Value:** `v=spf1 include:resend.com ~all`
   - **TTL:** `600` (или Auto)
3. Нажмите **"Save"**

**Важно:** Если у вас уже есть SPF запись, объедините их:
```
v=spf1 include:_spf.google.com include:resend.com ~all
```

#### B. DKIM записи (CNAME)

Resend предоставит несколько CNAME записей. Для каждой:

1. Нажмите **"Add"** в разделе **CNAME Records**
2. Заполните:
   - **Name/Host:** `resend._domainkey` (или другое значение от Resend)
   - **Value:** [значение от Resend, например: `resend._domainkey.resend.com`]
   - **TTL:** `600` (или Auto)
3. Нажмите **"Save"**

Повторите для всех CNAME записей, которые показывает Resend.

#### C. DMARC запись (TXT) - опционально

1. Нажмите **"Add"** в разделе **TXT Records**
2. Заполните:
   - **Name/Host:** `_dmarc`
   - **Value:** `v=DMARC1; p=none; rua=mailto:dmarc@faloclaro.com`
   - **TTL:** `600`
3. Нажмите **"Save"**

### 4. Проверка DNS записей

После добавления записей, проверьте их через онлайн инструменты:

1. **SPF проверка:**
   - https://mxtoolbox.com/spf.aspx
   - Введите: `faloclaro.com`

2. **DKIM проверка:**
   - https://mxtoolbox.com/dkim.aspx
   - Введите: `faloclaro.com`

3. **Общая проверка DNS:**
   - https://dnschecker.org
   - Введите домен и выберите тип записи

### 5. Верификация в Resend

1. Вернитесь в Resend Dashboard → Domains
2. Найдите домен **faloclaro.com**
3. Нажмите **"Verify"**
4. Дождитесь верификации (обычно 5-15 минут, максимум 48 часов)

### 6. Статус верификации

В Resend вы увидите:
- ✅ **Verified** - домен готов к использованию
- ⏳ **Pending** - ожидается верификация (проверьте DNS)
- ❌ **Failed** - проверьте правильность DNS записей

## Важные замечания

1. **Не удаляйте** существующие DNS записи, если они используются
2. **SPF запись** должна быть только одна - объедините с существующей, если есть
3. **DNS изменения** могут занять до 48 часов, но обычно работают за 5-15 минут
4. **Проверьте** все записи перед верификацией в Resend

## Пример существующих записей

Если у вас уже есть записи для других сервисов (например, Google Workspace), объедините их:

**SPF (если есть Google):**
```
v=spf1 include:_spf.google.com include:resend.com ~all
```

**MX записи** (не трогайте, если используются для получения почты):
```
Оставьте существующие MX записи без изменений
```

## Troubleshooting

### Проблема: DNS записи не обновляются

1. Очистите DNS кеш: https://www.whatsmydns.net/#TXT/faloclaro.com
2. Подождите 15-30 минут
3. Проверьте через разные DNS серверы

### Проблема: Верификация не проходит

1. Убедитесь, что все записи добавлены правильно
2. Проверьте, что нет опечаток в значениях
3. Убедитесь, что TTL не слишком большой (используйте 600)
4. Проверьте через mxtoolbox.com

### Проблема: Письма не отправляются

1. Убедитесь, что домен верифицирован в Resend
2. Проверьте, что используете правильный email адрес: `noreply@faloclaro.com`
3. Проверьте логи в Resend Dashboard → Logs
